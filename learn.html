getting started



You should start by playing with the built-in demo. Edit it as much as possible. And often launch the preview tab.

You cannot save the edition on the demo itself. What you do is press the save as button, creating another definition/webpage.

Hint: Using different background colors for each element is a great way to understand your layout!




opinionated HTML



The body of the webpage only contains directly frames and code remarks.

A frame is a 'div' tag which width is always 100% of the browser window width. When the ID of the frame is 'header' ('main' or 'footer'), then instead of a 'div' tag, a 'header' ('main' or 'footer') tag is created.




opinionated CSS



WAYW has its own way of doing CSS.

For starters, WAYW embeds a CSS reset sheet where all 'divs' display is inline-block and HTML elements are box-sizing: border-box.

You don't apply CSS on tags or IDs. Just on classes.

There are 10 kinds of classes: color, bg color, display, font, height, margin, overflow, padding, width and X class.

keyframe is part of the CSS in WAYW but it is not a class.

X class is the kind of class that you use to create what is missing in the other classes. Also, as WAYW writes this class at the end, what you write in it overrides all the other classes!

WAYW makes media query very easy to use.

There are some restrictions in the use of the classes. For example, you cannot set width to a frame. Also, no class can be set to a text (must set on its container).




memory



You can save your progress in the memory of the browser (not for anonymous mode), as long as the memory is not already full.




load & download



After you have finished editing, you can download the definition (.json file) and the webpage (.html file).

The .json file is your backup. This is the file you can load to WAYW in any browser/device.

Actually, what happens is a pseudo-download because your data never goes to the server.

In case your browser doesn't handle well the pseudo-download, you can display the data to copy and paste into a raw  text file. raw  means no possibility of text formatting (or else the data will be corrupted).




editing



When the selected button in the node tree is red, you can click it to make further edits, such as editing the text of a button.

During cut/copy & paste, WAYW takes care of all details, such as protecting against pasting on invalid location, ID conflicts and duplicating or overriding CSS classes (except for keyframes).

You can cut/copy & paste between different definitions/webpages!

Hint: If your webpage is too big. You can cut and paste some part into a blank webpage, edit that part there and cut and paste back!




hotkeys



Most sections of WAYW respond to hotkeys.

ENTER  moves to the next section. Says "OK".

ESCAPE moves to the previous section Says "Cancel".

SPACEBAR opens and closes the result preview tab.




images



PREVIEW TAB

For your security, browsers do not allow an externally hosted webpage (www.webasyouwish.com for example) to automatically open any file on your computer, including image files.

Due to this restriction, the preview tab cannot show any image located in your computer. That is the reason why there are two addresses in the image section. One for the preview tab and the other for the resulting webpage.

You must fill the address field of the image for the preview tab with the address of the image hosted somewhere in the web. For example:

- pinterest.com
- deviantart.com
- postimages.org

Important: You must use the address of the image; NOT the address of the page that contains the image. Maybe you will need to right-click the image and choose "copy image address".


RESULTING WEBPAGE

The downloaded .html file can open normally any image in your computer that is in the same directory, or in a subdirectory, as long as the image address/path is correct.




gotchas



The preview tab may not match exactly the resulting webpage, and it may be corrupted by your JavaScript code.

Because currently WAYW has little control about custom keyframes, they are not part of any cut/copy & paste. You have to edit manually.

Be careful when converting (apply) **temporary** color or bg color, because both share the same data. In other words, NEVER use **temporary** for color and bg color at the same time.

For safety reasons, sometimes pressing ENTER  has no effect. For example, when answering if it is OK to delete something.
